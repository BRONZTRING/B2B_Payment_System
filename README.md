🌍 B2B Cross-Border Blockchain Payment & AML Settlement System

基于区块链智能合约、高并发微服务与无监督学习的下一代跨国 B2B 支付与反洗钱 (AML) 沙盒系统。

📖 项目概述 (Overview)

在全球多极化与“去美元化”趋势下，本项目构建了一条绕过传统 SWIFT 高壁垒网络的极速“金融走廊”。
系统利用 Web3 智能合约实现去中心化资金担保，采用 Go 协程构建高并发 L2 路由撮合引擎，并独创性地将孤立森林 (Isolation Forest) 无监督学习算法降维部署于内存态，实现了极高吞吐量下（1100+ TPS）的毫秒级零误杀智能反洗钱 (AML) 拦截。

🚀 核心架构与技术亮点 (Core Features)

⛓️ 协议层：去中心化信任状态机

基于本地 Anvil EVM 运行的 PaymentEscrow 智能合约。

支撑买卖双向资金担保、物流链上确权以及跨国贸易仲裁 (Dispute Resolution)。

⚡ 中枢层：极速高并发路由引擎

基于 Go/Gin 构建的微服务，单机无锁化设计，突破跨语言 RPC 瓶颈。

采用 SQLite (WAL 预写式日志模式)，实现单文件底层数据库的高频并发状态沉淀。

🧠 免疫层：AI 反洗钱与 XAI 审计

纯 CPU 算力驱动的孤立森林算法，精准识别资金极值外逃与微结构洗钱 (Smurfing)。

引入 XAI (可解释性 AI) 特征归因机制，生成可视化的风控报告，支持高级管理员“人机协同 (HITL)”强制解锁。

🌐 可视化层：零 GPU 损耗战术雷达

Next.js + TailwindCSS 构建的玻璃拟态指挥大屏，支持中/俄 (ZH/RU) 双语。

纯数学贝塞尔曲线 + SVG 动态渲染全球 15 大战略港口 P2P 资金飞线，完美消除 WebGL 性能抢占。

source .venv/bin/activate

# 发起 50,000 笔并发交易
python load_test_bot.py


💡 此时观察前端大屏的 Telemetry 面板，您将看到 TPS 指针狂飙的视觉震撼效果。

TrustPay: Архитектура высоконагруженных распределенных систем (B2B Блокчейн-сеть) 🌐

Квалификационная работа (ВКР): «Архитектура высоконагруженных распределенных систем: оптимизация пропускной способности и потоковый анализ аномалий (на примере блокчейн-сети В2В-платежей)»

Данный репозиторий содержит исходный код прототипа системы трансграничных B2B-платежей следующего поколения. Проект объединяет технологию смарт-контрактов (EVM), высокопараллельную маршрутизацию (Goroutines) и алгоритмы машинного обучения без учителя (Isolation Forest) для создания децентрализованной, устойчивой к цензуре и защищенной от отмывания денег (AML) клиринговой сети.

🔬 Ключевые научные инновации (Научная новизна)

Преодоление проблемы I/O блокировок (I/O Lock Bottleneck): Использование СУБД SQLite в режиме WAL (Write-Ahead Logging) в связке с пулом горутин (Goroutine Pool) для достижения высокой пропускной способности (TPS) при записи состояний распределенного реестра без необходимости развертывания тяжеловесных СУБД (например, PostgreSQL).

Дистилляция моделей и эвристическая редукция (Model Distillation): Переход от ресурсоемкого онлайн-инференса ML-моделей к извлечению весов признаков (Feature Importance) с последующим их внедрением в пайплайн Go в виде эвристических правил в оперативной памяти. Это снижает задержку анализа аномалий с 45 мс до 0.4 мс (минимизация вычислительных накладных расходов).

Конечный автомат смарт-контракта (FSM Smart Contract): Реализация логики бизнес-процессов (условное депонирование, DeFi-факторинг, логистическое отслеживание и арбитраж) на базе децентрализованной виртуальной машины Ethereum (EVM).

🏗 Архитектура системы (Системный дизайн)

Проект состоит из трех независимых модулей, каждый из которых решает специфическую инженерную задачу:

1. contracts/ — Слой консенсуса и состояния (Blockchain Layer)

Ядро системы, отвечающее за неизменяемость транзакций.

Стек: Solidity ^0.8.20, фреймворк Foundry.

Функционал:

PaymentEscrow.sol: Смарт-контракт условного депонирования (эскроу). Поддерживает статусы: CREATED, PAID, SHIPPED, COMPLETED, DISPUTED.

DeFi-Факторинг: Возможность для экспортера получить 80% от суммы контракта в виде мгновенного кредита сразу после загрузки накладной (B/L) в блокчейн (requestFinancing).

Локальная сеть: Запускается на локальном узле Anvil (EVM).

2. backend/ — Слой высоконагруженной маршрутизации (Concurrency Gateway)

Связующее звено между пользовательскими интерфейсами и медленным блокчейн-консенсусом.

Стек: Golang (Gin), SQLite (WAL), go-ethereum (взаимодействие с RPC).

Функционал:

Управление 100 синтетическими профилями транснациональных корпораций.

Асинхронная постановка транзакций в очередь и взаимодействие со смарт-контрактами (обработка подписей, управление Nonce).

AML Engine: Механизм потокового скоринга на основе эвристики случайного леса (Random Forest). Транзакции с оценкой риска > 0.80 блокируются немедленно.

3. frontend/ — Пользовательский и административный интерфейсы (Client & Telemetry)

Стек: Next.js (App Router), React, Tailwind CSS, Viem (интеграция с Web3).

Узлы доступа:

/ (Business Portal): Интерфейс для импортеров и экспортеров. Позволяет инициировать переводы, запрашивать DeFi-кредиты и отслеживать перемещение грузов в рамках смарт-контракта.

/simulation (NOC Dashboard): Панель глобальной телеметрии ("Режим Бога"). Визуализация потоков транзакций (P2P), динамики блокировок ИИ и состояния конечного автомата в реальном времени.

📊 Экспериментальное исследование (DOOMSDAY Test)

Для оценки надежности архитектуры был разработан скрипт нагрузочного тестирования load_test_bot.py. Он генерирует синтетический поток транзакций (включая сценарии атаки и отмывания денег) для проверки пропускной способности (TPS) и уровня ошибок классификации (Байесовский риск).

Примечание для проверяющих: 17 академических графиков (включая анализ матрицы ошибок, ROC-кривую, задержку микросервисов и ECDF), использованных в 4-й главе диссертации, генерируются скриптом generate_offline_charts.py путем прямого парсинга БД SQLite после проведения нагрузочного теста. Все графики сохранены в директории /thesis_exports.
